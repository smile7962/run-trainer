💡 문제 유형에 따라 알고리즘과 평가지표가 달라진다!
⚙️ 모델의 핵심 구조 3/3
X (입력·독립변수·Feature)
↓
f(모델) = 수학적 함수
↓
y (출력·종속변수·Target)
  • 입력값 X 독립변수 / Feature — 모델 학습에 사용되는 변수
  • 출력값 y 종속변수 / Target — 예측하려는 값
  • 모델 f y = f(X) 를 만족하는 함수를 찾는 것이 모델링의 목표
🎯 중심경향 — 평균 / 중앙값 / 최빈값 1/5

예시 데이터: 1, 1, 1, 1, 2, 3, 3, 5, 10

  • 평균 Mean 모든 값의 합 ÷ 개수
    (1+1+1+1+2+3+3+5+10) ÷ 9 = 3
  • 중앙값 Median 오름차순 정렬 후 정중앙 값
    → 2
  • 최빈값 Mode 가장 자주 등장하는 값
    → 1 (4번 등장)
💡 이상치가 있으면 평균은 왜곡됨 → 중앙값이 더 대표적
📐 분산 & 표준편차 — 데이터의 퍼짐 2/5
  • 분산 Variance 각 값과 평균의 차이를 제곱하여 평균
    값이 클수록 데이터가 넓게 퍼짐
    예: 7.78
  • 표준편차 Std Dev 분산의 제곱근 → 원래 단위로 해석 가능
    √7.78 = 2.79
분산 = Σ(xᵢ - 평균)² ÷ n
표준편차 = √분산
💡 표준편차가 클수록 데이터 변동 폭이 크다
📦 분포 지표 — 최솟값 / 최댓값 / 사분위수 3/5
  • 최솟값 Min 데이터에서 가장 작은 값
  • 최댓값 Max 데이터에서 가장 큰 값
  • Q1 (25%) 전체 데이터 중 하위 25% 지점
  • Q2 (50%) = 중앙값 (Median)
  • Q3 (75%) 전체 데이터 중 하위 75% 지점
  • IQR Q3 - Q1 (중간 50% 범위, 이상치 탐지에 사용)
〰️ 왜도 (Skewness) — 분포의 비대칭 4/5
왜도 > 0 (양수)
오른쪽 꼬리 ▶

오른쪽에 집중
왜도 = 0
정규분포 (좌우대칭)
💡 왜도>0 이면 평균이 중앙값보다 크다 (극단값이 오른쪽에 있어 평균을 끌어올림)
🔗 상관계수 (Correlation Coefficient) 5/5

두 변수 간 관계를 수치화. 범위: -1 ~ +1

  • +1에 가까움 강한 양의 상관 — 한 변수↑ 다른 변수↑
  • -1에 가까움 강한 음의 상관 — 한 변수↑ 다른 변수↓
  • 0에 가까움 상관관계 거의 없음
양의 상관 ↗ 음의 상관 ↘ 무상관 —

🔧 데이터 전처리 4단계 1/5

컴퓨터가 데이터를 더 잘 이해할 수 있는 형태로 변환하는 과정

  • 1
    결측치 처리누락된 데이터 처리
  • 2
    이상치 처리극단적으로 벗어난 값 처리
  • 3
    인코딩 (Encoding)문자 → 숫자로 변환
  • 4
    스케일링 (Scaling)숫자 범위 통일
❓ 결측치 처리 (Missing Value) 2/5

데이터가 누락되어 비어있는 상태 (NaN)

  • ① 열 제거 결측치가 너무 많아 의미 없는 변수(컬럼) 삭제
  • ② 행 제거 결측치를 포함한 데이터 행 삭제
  • ③ 대체 평균 / 중앙값 / 최빈값 / 상수 등으로 채우기
    수치형→평균/중앙값, 범주형→최빈값
⚠️ 결측치 많으면 모델 예측 결과가 왜곡될 수 있음
⚡ 이상치 처리 (Outlier) — IQR 기법 3/5

일반적인 분포에서 크게 벗어난 값

IQR = Q3 - Q1
상단 경계 = Q3 + 1.5 × IQR
하단 경계 = Q1 - 1.5 × IQR

경계 밖의 값 = 이상치

  • ① 제거 이상치가 있는 행 삭제 (분석에 방해될 때)
  • ② 대체 상단/하단 경계값 또는 특정 상수로 변환
🔢 인코딩 (Encoding) — 문자→숫자 4/5
레이블 인코딩
범주를 정수로 변환
서울→2, 대전→0, 부산→1

⚠️ 문제: 컴퓨터가 2>0이라고 크기 순서로 오해 가능
원-핫 인코딩
범주마다 0/1 컬럼 생성
서울→[1,0,0]
대전→[0,1,0]
부산→[0,0,1]

⚠️ 범주 많으면 차원이 폭발적으로 증가
💡 순서 없는 범주형 → 원-핫 / 순서 있는 범주형 → 레이블
⚖️ 스케일링 (Scaling) — 숫자 범위 통일 5/5

변수 간 크기 차이로 인한 불공평한 학습 방지

Min-Max 정규화
범위를 0~1로 변환
x' = (x - min) / (max - min)

✅ 분포 보존
⚠️ 이상치에 민감
표준화 (Standard)
평균=0, 표준편차=1로 변환
x' = (x - μ) / σ

■■■■■■■ Train Set — 모델 학습용
■■ Validation Set — 학습 중 검증
■ Test Set — 최종 평가
  • 과적합 Overfitting 훈련 데이터에만 너무 최적화 → 새 데이터에서 성능 급락
  • 과소적합 Underfitting 훈련 데이터도 제대로 학습 못한 상태
💡 Train/Test는 반드시 겹치지 않게 분리!
📂 분류 모델 평가지표 3/5
혼동행렬 (Confusion Matrix)
실제 Positive실제 Negative
예측 P TP (맞음) FP (틀림)
예측 N FN (틀림) TN (맞음)
  • Accuracy 정확도 = (TP+TN) / 전체 — 클수록 좋음
  • Recall 재현율 = TP / (TP+FN)
    실제 P 중 P로 맞춘 비율 — 클수록 좋음
  • Precision 정밀도 = TP / (TP+FP)
    P로 예측한 것 중 실제 P 비율 — 클수록 좋음
  • F1-Score Recall + Precision 조화평균
    = 2 × (P×R) / (P+R) — 클수록 좋음
📉 회귀 모델 평가지표 4/5
  • MAE 평균 절대 오차
    실제값 - 예측값 절댓값의 평균
    작을수록 좋음
  • MSE 평균 제곱 오차
    오차를 제곱하여 평균 → 큰 오차에 더 민감
    작을수록 좋음
  • RMSE √MSE — 오차를 원래 단위로 해석
    작을수록 좋음
  • R² Score 결정계수 / 설명력
    모델이 데이터를 얼마나 잘 설명하는가
    1에 가까울수록 좋음
⚠️ 분류지표(Accuracy)는 클수록 좋고, 회귀지표(MAE/MSE/RMSE)는 작을수록 좋다! 혼동 주의
